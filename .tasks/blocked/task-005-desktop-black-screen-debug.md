# Task 005 — BLOCKED — Отладка черного экрана desktop UI

## Статус
Blocked after escalation protocol.

## Что известно
- Пользователь подтвердил, что после rebuild/reinstall `openkanban-desktop.exe` экран всё ещё черный.
- Предыдущий фикс `packages/desktop/vite.config.ts` с `base: './'` не устранил симптом.
- Полный тестовый набор проходит: `151 passed, 10 skipped`.

## Что было предпринято
1. Создан bounded debug task: `.tasks/in-progress/task-005-desktop-black-screen-debug.md`.
2. Запущен `frontend`-агент на систематическую локализацию root cause.
3. Retry 1: повторный запуск `frontend`-агента с ещё более узкой задачей на сбор диагностик без фиксов.
4. Retry 3: эскалация на `qa`-агент для evidence-based triage.

## Результат попыток
- Все 3 подагентных запуска вернули пустой handoff без обязательного блока `## Result`.
- По контракту orchestrator это считается `failed`.
- Из-за отсутствия аудируемых результатов ветка отладки не может продвигаться автоматически дальше без нарушения процесса.

## Почему задача заблокирована
- Нужны прямые runtime-доказательства из desktop WebView/Tauri runtime.
- В текущем автоматическом пайплайне подагенты не вернули ни локализацию, ни логи, ни безопасный фикс.
- Продолжать без доказательств означало бы гадать, что запрещено режимом systematic debugging.

## Что нужно для разблокировки
Один из следующих артефактов:
- скриншот окна + описание, есть ли курсор/реакция на клавиши;
- runtime logs из Tauri/WebView;
- запуск debug/dev режима desktop с доступной консолью ошибок;
- или ручная локальная диагностика через DevTools / WebView console.

## Следующий рекомендуемый шаг
- Выполнить ручную диагностику desktop runtime в debug-режиме и собрать console/runtime errors, затем открыть новый bounded task на основании этих артефактов.
